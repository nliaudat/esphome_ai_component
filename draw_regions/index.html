ESP32 AI Configuration Tool

Interactive tools to configure camera ROI, digit zones, and analog dials for ESPHome

How it works

This toolset helps you visually configure your ESP32 camera pipeline for meter reading and analog dial detection. Each tool is independent and optional — use only what you need for your specific board configuration.

All tools generate ready-to-use ESPHome YAML configuration that you can copy directly into your device config.

Optional

Set Camera ROI

Define the active area of the camera sensor. A smaller ROI acts as a digital zoom, improving frame rate and AI processing performance.

  • Select sensor model (OV2640, OV3660, OV5640)
  • Set camera rotation (0°–270° or custom angle)
  • Click and drag to draw the ROI rectangle
  • Optionally enforce 4:3 aspect ratio
  • Generates camera_window YAML block
🔧 Open ROI Tool
Optional

Set Digit Zones

Mark rectangular regions where digits appear on the meter display. These are passed to the TensorFlow Lite Micro model for digit recognition.

  • Draw green boxes around each digit
  • Order boxes in reading direction (left to right)
  • Coordinates are relative to the ROI output
  • Generates crop_zones for the meter reader
🔧 Open Digit Zones Tool
Optional

Set Analog Dials

Define circular regions for analog gauge/clock-dial detection. Includes a live preview pipeline and per-dial parameter tuning.

  • Draw orange squares around dials (center + drag outwards)
  • Live stage previews (grayscale, CLAHE, top-hat, etc.)
  • Multiple detection algorithms (radial profile, Hough, template match)
  • Per-dial parameter tuning (scan radius, angles, deadzone, etc.)
  • Generates analog_reader YAML block
🔧 Open Analog Dials Tool
